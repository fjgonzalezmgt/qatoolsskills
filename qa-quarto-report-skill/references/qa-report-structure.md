# QA Report Structure

Use this structure for deliverable QA reports.

## Core Sections

1. Title block: analysis type, product/process, author, date, revision.
2. Executive summary: decision, key evidence, risk, recommended action.
3. Scope and objective: what question the report answers.
4. Data readiness: source, date range, sample size, exclusions, quality issues.
5. Method: selected method, packages, assumptions, and why appropriate.
6. Results: tables, figures, models, indices, or priority rankings.
7. QA interpretation: what the result means for the process.
8. Risks and limitations: missing data, violated assumptions, sample size, stability, MSA, or confounding.
9. Actions and follow-up: containment, correction, confirmation, control plan, SPC, capability, or DOE follow-up.
10. Reproducibility appendix: session info, package versions, render info.

## Analysis-Specific Emphasis

- SPC: chart selected, baseline period, limits, signals, stability decision, reaction plan.
- MSA: study design, GRR or agreement results, acceptance for intended use, measurement risk.
- Capability: specs, observed conformance, stability, distribution, indices, PPM, action.
- FMEA/control plan: top risks, control gaps, recommended actions, linked control plan updates.
- CAPA: verified cause, corrective action, preventive action, effectiveness criteria.
- Pareto/AQL: defect priority, sampling plan, acceptance rule, OC/risk interpretation.
- DOE: objective, design, randomization, model, effects, diagnostics, optimum, confirmation.

## Executive Language

Use short, decision-oriented statements:

- "The process is stable for the baseline period analyzed."
- "Capability is exploratory because time-order stability was not demonstrated."
- "Measurement variation is too high for capability decisions."
- "The recommended setting requires confirmation runs before release."
- "Sampling accepted this lot, but does not demonstrate process capability."
