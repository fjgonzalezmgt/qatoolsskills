# FMEA And Control Plan Guide

## FMEA Logic Checks

- Function should describe what the process or product must do.
- Failure mode should describe how the function can fail.
- Effect should describe impact to customer, next process, safety, compliance, or performance.
- Cause should be specific enough to act on.
- Prevention controls should reduce occurrence by controlling the cause.
- Detection controls should find the cause or failure mode before escape.
- Recommended actions should reduce severity, occurrence, or detection risk and include owner and due date.

## Rating Discipline

- Use the organization's rating manual when available.
- Do not compare RPN across different rating systems.
- Treat high severity items seriously even when occurrence is low.
- If AIAG-VDA action priority is used, do not replace it with RPN unless the user requests it.

## Control Plan Linkage

Each critical or high-priority risk should map to:

1. process step or operation;
2. product or process characteristic;
3. specification or acceptance criterion;
4. measurement or control method;
5. sample size and frequency;
6. control owner;
7. record or evidence;
8. reaction plan for nonconformance or loss of control.

## Common Gaps

- Detection control described as prevention control.
- Cause written as "operator error" without mechanism.
- Reaction plan says only "notify quality" without containment or disposition.
- Special characteristic not reflected in control frequency.
- High-severity risk closed without objective evidence.
